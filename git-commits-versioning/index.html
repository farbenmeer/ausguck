Git commits

Simple

<type>: <subject>

feat: Add age

Extended

<type>(<scope>): <subject>
<BLANK LINE>
<body>

feat(user): Add age

Add age field to all premium users.

Rules - type

💍 testAdding missing tests
🎸 featA new feature
🐛 fixA bug fix
🤖 choreBuild process or auxiliary tool changes
✏️ docsDocumentation only changes
💡 refactorA code change that neither fixes a bug or adds a feature
💄 styleMarkup, white-space, formatting, missing semi-colons...

Rules - subject

☝️ImperativeUse the imperative mood
-> written as if giving a command or instruction
👼Short subjectMax. 50 character subject
🔊CapitalizeFirst letter in subject
🧐No periodAt the end of the subject
⏱PresentNo future no past

❌ Don'ts ❌

feat(user): New age field added 

feat(user): added age 

feat(user): Add age. 

feat(user): Adds age  

feat(user): Add age to the profiles and other sides

MINOR version when you add functionality in a backwards-compatible manner
PATCH version when you make backwards-compatible bug fixes
